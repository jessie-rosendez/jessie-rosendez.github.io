Key Design Choices
Advisor style profile is extracted once during onboarding, stored as JSON in Supabase, and injected into Agent 1 at generation time. Every outreach email sounds like the advisor, not the system. FastAPI and the React frontend are fully decoupled. Agent runs, outreach messages, and activity logs are all persisted independently so the advisor can inspect every decision the system made.

At single-advisor scale the system works immediately. The harder problem, surfaced during evaluation at a top wealth management firm, is what happens at firm-wide scale: one trigger fires, multiple advisors generate outreach to the same prospect. The allocation engine solves that.
Prospect LockingFirst advisor to claim a prospect holds an exclusive window (30–90 days, configurable). No other advisor at the firm can target the same person.
Proximity RoutingWhen a trigger fires, the system scores advisors by geography, AUM fit, and industry expertise before assigning the lead. Closest match gets first right of refusal.
Compliance Audit TrailEvery contact is logged: advisor ID, timestamp, classification, action taken. Full history available for compliance review without touching firm systems.
Intelligence Layer DesignWeak-signal detection across 25 precursor event types: governance transitions, UCC filings, estate structuring. Entity graph resolution and temporal clustering to surface opportunities before public confirmation.

Key Design Choices
Each agent receives the full prior context chain, not just its immediate input. SequentialBuilder from the agent-framework library orchestrates the handoff. AzureCliCredential handles secure authentication without hardcoded keys. Python AsyncIO enables non-blocking execution across the pipeline.

MCP Design Pattern
The agent does not have direct database access. Instead it calls tools exposed by the MCP server, which act as a controlled interface to the underlying data. This pattern mirrors how enterprise systems enforce separation of concerns: the AI reasons and decides, the tools execute and return data.

The MCP server acts as a controlled interface between the AI and live business data. Enterprises can expose any internal data source as MCP tools without giving the model direct database access. This makes the agent auditable, secure, and easy to extend.

Key Design Note
The agent uses Pydantic for tool parameter validation, ensuring the model passes correctly typed and structured data before any action executes. The approval mode setting controls whether the agent requires human confirmation before submitting, making it easy to toggle between fully autonomous and supervised operation.
